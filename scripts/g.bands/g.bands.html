DESCRIPTION

Prints band references information related to image collections.

Band reference (a shortcut or fully defined band reference identifier) is specified by band option. Extended metadata is printed only when -e flag is given.

Image collections

Image collections are the common data type to reference time series of multi band data. It is used in many frameworks (see Google Earth Engine API for example) to address multi spectral satellite images series. GRASS supports a multi-band raster layer approach rudimentarily with the imagery group concept (i.group). A new band reference concept is designed in order to support image collections in GRASS GIS.

NOTES

Specific band reference can be assigned to a raster map by i.band module.

Band reference concept is supported by temporal GRASS modules, see t.register, t.rast.list and t.info modules for real examples.

Band reference registry files

Band reference information is stored in JSON files with a pre-defined internal data structure. A minimalistic example is shown below. 
{
    "Sentinel2A": {
        "description": "The Sentinel-2 A bands",
        "shortcut": "S2A",
        "instruments": "optical and infrared",
        "bands": {
            "1": {
                "central wavelength (nm)": 442.7,
                "bandwidth (nm)": 21
            },
            "2": {
                "central wavelength (nm)": 492.4,
                "bandwidth (nm)": 66
            }
}
Each series starts with an unique identifier ("Sentinel2A" in example above). Required attributes are only two: a shortcut and bands. Note that a shortcut must be unique in all band reference registry files. Number of other attributes is not defined or even limited (in example above only description and instruments attributes are defined). List of bands is defined by a bands attribute. Each band is defined by an identifier ("1", "2" in example above). List of attributes describing each band is not pre-defined or limited. In example above each band is described by a central widthlength (nm) and bandwidth (nm).

Band reference identifier defined by band option is given by a shortcut in order to print band reference information for whole series or by specific shortcut_band identifier.

System-defined registry files are located in GRASS GIS installation directory ($GISBASE/etc/g.bands). Note that currently g.bands allows to manage only system-defined registry files. Support for user-defined registry files is planned to be implemented, see KNOWN ISSUES section for details.

EXAMPLES

Print all available band references

g.bands

S2A_1
S2A_2
...
S2B_1
S2B_2
...
Ls7_1
Ls7_2
...

Filter band references by a shortcut

Only band identifiers related to Sentinel-2A satellite will be printed. 
g.bands band=S2A

S2A_1
...
S2A_12
Similarly for both Sentinel-2 satellites. 
g.bands band=S2

S2A_1
...
S2A_12
S2B_1
...
S2B_12

Print extended metadata for specified band identifier

Extended metadata related to the first band of Sentinel-2A satellite will be printed. 
g.bands -e band=S2A_1

description: The Sentinel-2 A bands
shortcut: S2A
instruments: optical and infrared
    band: 1
        central wavelength (nm): 442.7
        bandwidth (nm): 21

KNOWN ISSUES

g.bands has currently very limited functionality. Only system-defined band references are supported. The final implementation will support managing (add, modify, delete) user-defined band references.

Only very limited number of band references is currently implemented. Namely Sentine2-A, Sentinel2-B and Landsat7 sattelites. This will be improved in the near future.
