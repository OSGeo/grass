DESCRIPTION

g.subproject changes the current working MAPSET, LOCATION, or GISDBASE. This is a fairly radical maneuver to run mid-session, take care when running the GUI at the same time.

With g.subproject, the shell history (i.e. .bash_history file of the initial project will be used to record the command history.

NOTES

By default, the shell continues to use the history for the old subproject. To change this behaviour the history can be switched to record in the new subproject's history file as follows: 
# bash example
history -w
history -r /"$GISDBASE/$LOCATION/$MAPSET"/.bash_history
HISTFILE=/"$GISDBASE/$LOCATION/$MAPSET"/.bash_history

EXAMPLES

Print the name of the current subproject

To print the name of the current subproject, use the -p command as shown below: 
g.subproject -p

List available subprojects

To list available subprojects, use the -l command as shown below: 
g.subproject -l
This should list all the subprojects, such as: "landsat new PERMANENT user1."

Change the current subproject

To change the current subproject to "user1" use the following command: 
g.subproject subproject=user1 project=nc_spm_08_grass7
You should receive the following message: "Subproject switched. Your shell continues to use the history for the old subproject."

Create a new subproject

To create a new subproject, use the -c tag as shown below: 
g.subproject -c subproject=new project=nc_spm_08_grass7
