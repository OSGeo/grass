DESCRIPTION

This program allows the user to control access to the current subproject. Normally, any user can read data from any GRASS subproject. But sometimes it is desirable to prohibit access to certain sensitive data. The g.access command allows a user to restrict read and execute access to the current subproject (see UNIX chmod command). g.access will not modify write access to the current subproject.

The user may, for example, allow only users in the same UNIX group to read data files in the subproject, or restrict the subproject to personal use only.

NOTES

Under GRASS, access to the subproject PERMANENT must be open to all users. This is because GRASS looks for the user's default geographic region definition settings and the project TITLE in files that are stored under the PERMANENT subproject directory. The g.access command, therefore, will not allow you to restrict access to the PERMANENT subproject.

The g.subprojects command isn't smart enough to tell if access to a specified subproject is restricted, and the user is therefore allowed to include the names of restricted subprojects in his search path. However, the data in a restricted subproject is still protected; any attempts to look for or use data in a restricted subproject will fail. The user will simply not see any data listed for a restricted subproject.

UNIX filesystem access controls and g.access actions are not supported by MS-Windows.
